# AGENTS.md — контекст проекта для AI-агентов

Этот файл — справочник для любого ассистента (Codex, Claude, Cursor и т.д.),
который правит этот репозиторий. Здесь зафиксированы соглашения проекта, где
что лежит и какие правки уже были внесены — чтобы при просьбе «поправь X» не
приходилось искать концы с нуля.

## 1. Что это за проект

Сайт-портфолио **levmich.studio**. Статика, деплой через GitHub Pages из
ветки `main`. Любой коммит в `main` → автодеплой, без билда.

Стек: чистый HTML/CSS/JS + GSAP + ScrollTrigger + Lenis (через CDN). Сборщика
нет.

## 2. Структура

```
index.html              ← главная (v3). Большой инлайн-<script> внутри
                          управляет мобильным/десктопным режимом и
                          v3-stacked сценами. Это НЕ дубль main-v2.js.
info.html               ← страница «Инфо» (договор оферты, политики)
case-front.html         ← кейс Front
case-three.html         ← кейс «Три семёрки»
case-wedding.html       ← кейс «Свадебный сайт»

Assets/styles/style.css ← единственный CSS на весь сайт
Assets/scripts/main-v2.js ← общий JS (включая i18n RU/EN, dock/menu, reveal,
                            services/portfolio scroll-stack, и т.д.)

Assets/SVG_files/       ← иконки (соцсети, стрелки, иконки навигации)
Assets/Main_page/, Assets/Case_*/  ← картинки (преимущественно .webp)
```

## 3. Cache-busting — ОБЯЗАТЕЛЬНО при изменении CSS/JS

CSS и JS подключены с query-параметром:

```html
<link rel="stylesheet" href="Assets/styles/style.css?v=116" />
<script defer src="Assets/scripts/main-v2.js?v=72"></script>
```

**Правило:** при правке `Assets/styles/style.css` бампать `?v=` **на всех 5
страницах сразу** (index, info, case-front, case-three, case-wedding) — иначе
у части пользователей закэшится старый CSS. То же для `main-v2.js`.

**Текущие версии (на 2026-06-02):** `css?v=116`, `main-v2.js?v=72`.

Удобный скрипт-однострочник в PowerShell:
```powershell
$enc = New-Object System.Text.UTF8Encoding($false)
foreach ($f in 'index.html','info.html','case-front.html','case-three.html','case-wedding.html') {
  $t = [IO.File]::ReadAllText($f)
  $t = $t.Replace('style.css?v=116','style.css?v=117')
  [IO.File]::WriteAllText($f, $t, $enc)
}
```

## 4. Где что лежит в style.css

Файл большой (~6 тыс. строк), но устроен предсказуемо: сверху — базовые
правила (десктоп), ниже — портретные/мобильные `@media`-блоки.

| Что | Где |
|---|---|
| Hero | базовые: `.hero`, `.hero__*` (около 300–600) |
| Услуги (v3 redesigned) — desktop | `.services`, `.service`, `.services__head` (~640–1080) |
| Услуги — мобайл | `@media (orientation: portrait)` блок ~2789–2950 |
| Портфолио (кейсы) — desktop | `.portfolio`, `.portfolio-case*` (~1180–1330) |
| Портфолио — мобайл | `@media (orientation: portrait)` блок ~2974–3055 |
| Dock / меню / overlay | `.dock`, `.dock__*`, `.menu-overlay` (~1623–2210) |
| Behance drop-in кнопка (на страницах кейсов) | `.dock__behance-*` (~1824–2025) |
| Футер | `.footer`, `.footer__*` (~1340–1580 + мобайл ~3076–3170) |
| Info-страница | `.info-*` (~3850+) |

**Мобильный портретный CSS большого блока услуг/портфолио лежит в одном
`@media (orientation: portrait)` около строки 2598–3536.** Если что-то «не
залипает только на мобайле» — искать там.

## 5. Где что лежит в main-v2.js

- **i18n словари** (RU/EN, по странице): ~строки 130–470
  - Главная: `dictRu` (~135), `dictEn` (~245)
  - Меню: внутри `nav.*` каждого словаря (`title`, `items`, `slogan`,
    `magnet`, `behancePrompt`, ...)
- **Применение текстов на страницу:** функция со списком `setText(...)` /
  `setHtml(...)` около строк 615–700.
  - ⚠️ **Если в словарь нужно положить `<br/>` или другой HTML — применять
    через `setHtml`, не `setText`.** Иначе теги отрисуются как текст. Уже
    есть прецедент: меню-слоган EN рендерили через setText, `<br/>` ушёл в
    обычный текст. Сейчас применяется через `setHtml('.dock__menu-slogan',
    d.nav.slogan)`.
- **Переключение языка:** `?lang=ru|en` + `localStorage`, при клике на
  `#dock-lang-btn` страница перезагружается (~735).
- **Reveal-анимации:** IntersectionObserver, `rootMargin: '0px 0px -10% 0px'`
  (~строка 900).
  - ⚠️ Элемент в самом низу страницы (например, `.footer__copyright`) с
    классом `reveal` НИКОГДА не появится: он постоянно сидит в исключённой
    нижней зоне. Решение — не вешать `reveal` на такие элементы. Уже было
    исправлено для ©.
- **Услуги/портфолио мобильный «стек»:** `initServicesStack()`,
  `initPortfolioStack()` (~1700, 1729). Простой scroll-trigger scale+opacity
  на следующую карточку.
- **`initMobilePanelStacks()`** (~1508) — старая v2-сцена стэка для
  портфолио. **На v3-главной отключена** (skip по
  `data-page-version="v3"`). Не реактивировать.
- **Inline-скрипт в `index.html` (~1923–3200)** — отдельный мини-фреймворк
  для v3 главной: переключение `landscape ↔ portrait` без перезагрузки,
  управление классами `v3-stacked-cases` / `v3-desktop`. Сейчас
  **`v3-stacked-cases` НЕ добавляется** (отключено намеренно — кейсы должны
  вести себя как услуги).

## 6. Языки

- RU — по умолчанию (`<html lang="ru">`).
- EN — переключается из `dock-lang-btn`. Текст подменяется через
  `dictEn`/`dictEnInfo`/`dictEnCases` в `main-v2.js`. Любая правка текста на
  главной/инфо/кейсах → **править нужно в обоих словарях**.

## 7. Картинки

- Формат — **WebP, качество 90** (визуально без потерь). На странице кейса
  есть desktop и mobile варианты (`<source media="(max-width: 640px)">`).
- Для конвертации используется Python (Pillow). На этой машине доступен
  именно `py` (Windows launcher) — не `python`. См. `levmich-deploy-and-tooling`.
- ⚠️ **EXIF-orientation:** при ресайве фото с телефона обязательно вызывать
  `ImageOps.exif_transpose(im)` перед `.save()`, иначе фото перевернётся
  (баг был с GOAT-фото). Pillow не запекает orientation по умолчанию.

## 10. Журнал значимых правок (последние сессии)

Кратко, что было сделано — чтобы при просьбе «верни как было» / «поправь
там, где ты делал X» можно было быстро найти место.

### Hero
- Убрана плашка `.hero__info` (десктоп и мобайл).
- На мобайле под фото — пилюля «Подробнее» с иконкой click-tap.

### iOS Safari
- Анимации работают и под `prefers-reduced-motion: reduce` (телефон владельца
  был с включённым reduce-motion — без этого ничего не двигалось).
- Behance drop-in: плавная анимация на всех устройствах (без GSAP-only).
- iOS: меню теперь выплывает при открытии (раньше прыгало).
- iOS: параллакс фото в GOAT восстановлен.
- iOS: кнопки-стрелки кейсов открывают кейс (раньше тап не доходил до `<a>`,
  потому что внутри был перехватчик курсора).

### Карточки (мобайл)
- **Анимация кейсов = анимации услуг:** убрали JS-сцену стопки и липкий
  заголовок, осталось простое перелистывание. `.portfolio__title` →
  `position: static` (раньше был `sticky; top:40px`).
- **Расстояние между карточками кейсов:** `margin-bottom: 40px` (раньше 100).
- **Внутри карточки:** фото → название = **10px**, название → теги = **10px**
  (`.portfolio-case` gap + `.portfolio-case__copy` gap, оба в мобильном
  блоке). До этого было 4/0 (слишком тесно), потом 14/14 (слишком воздушно).
- **Между последней карточкой и футером ≈ 80px:** убрал лишние
  `padding-bottom: 140px` у `.portfolio__grid` и 50px у `.portfolio`; wedding
  `margin-bottom: 0`. Остаётся внутренний `padding-top: 60px` футера +
  отступы заголовка футера = ~80–82px видимого зазора.

### Услуги
- Цены допов: оранжевая полупрозрачная пилюля-обводка
  (`.services__explain-price`).
- Заголовок секции на мобайле — НЕ sticky (`services__head` имеет sticky
  правило `top: 40px`, но фактически на портретном вьюпорте — `static` из-за
  специфики `services--redesigned`). При правке мобильных отступов услуг —
  это эталон, по которому подгоняли кейсы.

### Меню (dock)
- z-index: `.dock` = **1000**, `.menu-overlay` = **990**. Меню гарантированно
  на самом верхнем слое (`.case-cursor` имеет z-120).
- Меню-слоган EN перенесён на новую строку:
  `'We make it beautiful<br/>Ugly&nbsp;— not&nbsp;our&nbsp;thing'`.
  ⚠️ Применяется через `setHtml('.dock__menu-slogan', ...)` (НЕ setText) —
  иначе `<br/>` отрисуется как текст.

### Behance drop-in (на страницах кейсов)
- **Убран `filter: blur(7px)`** из базового состояния `.dock__behance-button`
  и `filter` из transition. Bē-знак больше не размыт в кадрах анимации.
- Анимация осталась мягкой за счёт `opacity` + `scale .64→1` + `translateY`.

### Футер
- Один и тот же футер на всех 5 страницах (разметка идентична).
- `© Levmich Studio 2026` — отдельный элемент `.footer__copyright`:
  - **Мобайл:** `position: static`, последний элемент футера → внизу.
  - **Десктоп:** `position: absolute; left:0; right:0; bottom:0; width:fit-content;
    margin-inline: auto` → строго по центру внизу. Раньше центрировали через
    `left:50%; translateX(-50%)` — давало смещение ~80px.
  - ⚠️ **У `.footer__copyright` НЕ должно быть класса `reveal`**, иначе
    IntersectionObserver не покажет его (он сидит в исключённой нижней зоне
    `rootMargin: -10%`). Уже снято на всех страницах.

### Инфо-страница
- Договор оферты обновлён под новые цены (см. секцию `2.3` про базовые цены
  услуг — суммы должны совпадать с услугами на главной).

### Изображения
- Все основные изображения уже WebP q90 (~50 МБ → ~3.9 МБ суммарно).
- Перевёрнутое EXIF-фото в GOAT исправлено (см. п. 7).
